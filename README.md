# 🏥 院内案内 WebAR デモ

病院の人間ドック受診者向け、iPhone Safari 対応の院内誘導 WebAR デモアプリです。

---

## ファイル構成

```
hospital-ar-guide/
├── index.html        # 受診者向け画面（iPhone Safari で使用）
├── admin.html        # スタッフ向け管理画面（PC ブラウザで使用）
├── markers.html      # マーカー印刷用シート
├── js/
│   └── data/
│       ├── patients.js     # 受診者データ（localStorage 管理）
│       └── locations.js    # 地点・マーカールール定義
└── README.md
```

---

## 起動方法

### ローカル確認（PC）

```bash
# Python 3 の場合
cd hospital-ar-guide
python3 -m http.server 8080

# Node.js の場合
npx serve .
```

ブラウザで以下を開く：
- 管理画面：http://localhost:8080/admin.html
- 受診者画面：http://localhost:8080/index.html

### iPhone Safari での確認（HTTPS 必須）

1. ngrok や Cloudflare Tunnel で HTTPS を用意する

```bash
# ngrok の場合
ngrok http 8080
```

2. 発行された HTTPS URL を iPhone の Safari で開く

## デモ手順

### 管理画面（スタッフ側・PC）

1. `admin.html` を開く
2. 受診者一覧が表示される（ダミーデータ：1001〜1005）
3. 受診者の「案内先を変更」で次の行き先を変更 → 保存
4. 受診者画面に自動で反映される（10秒ごとに自動更新）

### 受診者画面（iPhone Safari）

1. `index.html` を開く
2. 受診番号を入力（例：`1001`）
3. 次の案内先が表示される
4. 「マーカーをスキャンする」をタップ
5. 現在地のマーカーを選択
6. 矢印と案内テキストが大きく表示される

## iPhone Safari 制約

- **HTTPS 必須**：カメラ権限は HTTP では動作しない
- **フルスクリーン**：ホーム画面追加（Add to Home Screen）でより没入感が増す
- **自動更新**：バックグラウンド中は JS タイマーが停止することがある（デモ時は注意）

---

## 技術選定について

当初 Unity + AR Foundation を検討しましたが、**iPhone Safari での安定動作を最優先**とし、以下の理由で **MindAR.js + Three.js** を採用しました：

- CDN のみで動作、ビルド不要
- iPhone Safari 15+ で安定動作
- 画像マーカー認識（印刷物をそのままマーカーに）
- Three.js と組み合わせることで 3D 矢印も後から拡張可能

現バージョンは **マーカー選択方式**（実際のカメラ認識の代替）で動作します。
MindAR の `.mind` ファイルを用意することで、カメラによる実際の画像認識に拡張できます。
